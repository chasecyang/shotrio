# 积分系统定价更新 (2024)

## 📊 更新概览

将积分体系从 **1美金 = 1积分** 升级为 **1美金 = 10积分**，提供更灵活的定价策略和更好的用户体验。

## 🎉 更新优势

### 1. 定价灵活性
- 可以精确到1积分调整（如6、7、8积分）
- 方便做 A/B 测试和促销活动
- 未来可实现动态定价

### 2. 用户体验提升
- "900积分"比"90积分"看起来更丰富
- 大数字给人更好的价值感知
- 便于理解"1美金=10积分"的换算

### 3. 营销优势
- "仅需8积分"比"仅需0.8积分"更易传播
- "充值送3000积分"比"送300积分"更有冲击力
- 更好的促销标语潜力

### 4. 商业灵活性
- 保持健康利润率（41-51%图片，16-30%视频）
- 留有降价促销空间（可临时降到6积分）
- 未来可针对不同场景差异化定价

## 🚀 未来扩展建议

### 1. 音频开关功能（已预留代码）
当前所有视频都是带音频生成（20积分/秒），未来可以：
- 在UI中添加"生成音频"开关
- 无音频时使用 `CREDIT_COSTS.VIDEO_GENERATION_NO_AUDIO_PER_SECOND` (12积分/秒)
- 提升视频业务利润率到71%

### 2. 动态定价策略
- 新用户首次生成优惠（6积分/张）
- 批量生成折扣（第100张开始6积分）
- 会员专属价格（VIP 7积分/张）

### 3. 积分促销活动
- 双倍积分日
- 节日特惠（充值送50%）
- 推荐奖励机制

### 4. 成本优化
- 与供应商协商批量折扣
- 混合使用多个AI服务商
- 智能路由到性价比最高的服务

## ⚠️ 注意事项

### 数据迁移
本次更新**不涉及历史数据迁移**，因为用户确认无需考虑历史数据。

如果将来有历史用户，需要：
1. 运行迁移脚本将现有积分余额 ×10
2. 更新历史交易记录的 amount 字段 ×10
3. 通知用户积分体系升级

### 代码部署
所有修改已完成，可以直接部署：
```bash
# 检查构建
npm run build

# 部署到生产环境
npm run deploy
```

### 监控建议
上线后需要监控：
- 各套餐的购买转化率
- 图片 vs 视频的使用比例
- 平均单用户消费金额（ARPU）
- 整体利润率

根据数据及时调整定价策略。
